## 🧠 PLANNING & CONSISTENCY FEATURES

---

### 7. 📅 Weekly Review Mode

- Every **Sunday**, automatically show a full-screen "Week in Review" overlay when the app is opened.
- The review screen should display:
  - Total practice sessions logged this week
  - Skills practiced vs skipped
  - Longest streak achieved
  - Best performing skill of the week (most sessions logged)
  - Mood trend summary (average mood across the week — see Feature 13)
  - A motivational closing message like: *"Great week, [Name]! You practiced X times. Keep it up! 🚀"*
- Include a **"Set Next Week's Intentions"** section at the bottom: a simple text input where users write a short goal for the upcoming week. Store in localStorage as `pst_weekly_intention`.
- The weekly review card should have a premium feel — large typography, glowing border, animated stats that count up on mount.
- Add a **"View Past Reviews"** button that opens a history of all previous weekly reviews stored in `pst_weekly_reviews` (array of weekly summaries).
- If the user dismisses it, they can re-open it from the dashboard via a "📅 This Week's Review" button that appears only on Sundays.

### 8. 🗓️ Skill Scheduling

- When adding or editing a skill, allow the user to assign **specific practice days** (checkboxes: Mon / Tue / Wed / Thu / Fri / Sat / Sun).
- Store scheduled days inside the skill object as `scheduledDays: ["Mon", "Wed", "Fri"]`.
- The **Dashboard "Today's Targets"** section should only show skills scheduled for the current day of the week — reducing cognitive overload.
- On the Skill Card, display the scheduled days as small colored pill badges below the skill name.
- In the **Progress > Monthly Consistency Calendar**, mark days where a skill was scheduled but not practiced in a soft red/amber — distinguishing "scheduled miss" from "not scheduled".
- Add a **"This Week's Schedule"** mini-widget on the dashboard: a horizontal 7-day strip showing which skills are due each day. Clicking a day shows that day's skills in a tooltip/popover.

---

### 9. ⏱️ Focus Timer (Pomodoro)

- Add a **Focus Timer** accessible from each Skill Card via a timer icon button 🕐.
- Timer has two modes selectable via toggle:
  - **Pomodoro Mode**: 25 min work → 5 min break (repeat). Show cycle count.
  - **Custom Mode**: User sets their own duration (5–120 minutes via a slider).
- Timer UI should be a **large circular countdown ring** (SVG stroke-dasharray animation) with:
  - Time remaining in large digits at the center
  - Skill name shown below
  - Start / Pause / Reset controls
  - Session mode label (Focus / Break)
- When a focus session completes:
  - Play a soft chime sound (use Web Audio API to generate a tone — no external audio files needed).
  - Auto-log the practice session to that skill (duration = timer duration, notes = "Focus session via Pomodoro timer").
  - Show a small toast: *"Great focus session! Practice logged for [Skill Name] ✅"*
  - Increment streak if applicable.
- Track total Pomodoro sessions in localStorage as `pst_pomodoro_count`. Add a badge:
  - **"Deep Focus"** 🧘 — Complete 5 Pomodoro sessions total.
  - **"Flow State"** 🌊 — Complete 20 Pomodoro sessions total.
- The timer should continue running if the user navigates to another page (use a global timer context/state).
- Show a small **persistent timer pill** in the top navbar when a timer is active, so the user always sees the countdown.

## 🎮 GAMIFICATION FEATURES

### 10. ⭐ Skill XP & Levels

- Each skill now has an **XP (experience points)** system alongside the existing progress percentage.
- XP is earned by logging practice sessions:
  - Each practice session = **+10 XP base**
  - Bonus XP modifiers:
    - Session duration > 30 min = **+5 bonus XP**
    - Mood logged as "On Fire" 🔥 = **+3 bonus XP**
    - Completing a daily target = **+5 bonus XP**
    - Maintaining streak (3+ days) = **+2 bonus XP per session**
- XP thresholds for each level:

  | Level | Title | XP Required |
  |---|---|---|
  | 1 | Beginner | 0 XP |
  | 2 | Apprentice | 50 XP |
  | 3 | Skilled | 150 XP |
  | 4 | Advanced | 350 XP |
  | 5 | Expert | 700 XP |
  | 6 | Master | 1200 XP |

- On each **Skill Card**, show:
  - Current level title with a colored badge
  - XP progress bar toward next level (e.g., "230 / 350 XP to Advanced")
  - Level number in a small circle icon on the card corner
- When a skill levels up, trigger a **level-up animation**: the card border glows, a "LEVEL UP! ⭐" banner slides across the card, and a satisfying sound plays.
- Add new badges for leveling:
  - **"First Rank"** 🥉 — Any skill reaches Level 2
  - **"Rising Star"** 🌠 — Any skill reaches Level 4
  - **"Grandmaster"** 👑 — Any skill reaches Level 6

### 11. 🏁 Weekly Challenges

- Add a **"Challenges"** section on the dashboard (collapsible card) and a full **Challenges page** accessible from the sidebar (🏁 Challenges).
- Generate **3 new challenges every Monday** and store them in `pst_challenges`.
- Challenge types (rotate randomly):
  - *"Practice any skill for 5 days this week"*
  - *"Log 3 Pomodoro sessions in one day"*
  - *"Complete all daily targets on 3 different days"*
  - *"Practice a skill you haven't touched in 5+ days"*
  - *"Reach Level 3 in any skill this week"*
  - *"Log a practice session with mood 'On Fire' 🔥 3 times"*
  - *"Add a new skill and practice it at least twice"*
  - *"Achieve a 5-day streak"*
- Each challenge card shows:
  - Challenge description
  - Progress tracker (e.g., "3 / 5 days")
  - Reward badge preview (greyed out until earned)
  - Days remaining until Sunday
  - A glowing green **"COMPLETED"** stamp when done
- Completing a challenge awards a unique **Challenge Badge** with the week number (e.g., "Week 3 Champion 🏁").
- Store completed challenge history in `pst_challenge_history`.

### 12. ⚔️ Skill Duels (Friend Challenge)

- Add a **"Duel"** button on each Skill Card.
- When clicked, open a **"Create Duel"** modal where the user:
  1. Sets the skill being dueled
  2. Sets the duel duration (3, 5, or 7 days)
  3. Sets the goal metric (most sessions logged OR most XP earned)
  4. Clicks **"Generate Duel Link"** — creates a shareable URL like `?duel=abc123` with duel params encoded in the URL hash/query string (no backend needed — use URL encoding).
- When another user opens the duel link:
  - They see a **"You've been challenged!"** screen showing the challenger's stats.
  - They click **"Accept Duel"** to start tracking their side.
  - Both users track progress locally; the duel compares on honor system.
- On the **Achievements page**, show an **"Active Duels"** section listing ongoing duels with countdown timers.
- When duel time expires, show a **"Duel Result"** screen:
  - Winner gets a **"Duel Victor" ⚔️** badge.
  - Loser gets a **"Good Fight" 🤝** badge.
- Store duel state in `pst_duels`.

### 13. 🕐 Best Time of Day Analytics

- When logging a practice session, **automatically record the time of day** (hour) of the log.
- On the **Progress page**, add a new section: **"Your Peak Practice Hours"**.
- Display a **24-hour heatmap bar** (horizontal bar divided into 24 segments, colored by session frequency) showing when the user most often practices.
- Below the heatmap, show an insight card:
  - *"🌙 You practice most at 9 PM — that's your peak focus time!"*
  - *"⚡ Your reminder is set for 6 PM. Consider shifting it to 8:30 PM to match your natural rhythm."*
- Also show a breakdown per skill: each skill's most common practice hour.
- Store session timestamps in existing `pst_sessions` — add `loggedAt: ISO timestamp` field to each session object.

### 14. 📉 Consistency Score

- Calculate a **Consistency Score (0–100)** for each user, updated daily.
- Formula based on weighted factors:
  - Streak length (30% weight)
  - % of scheduled days practiced this month (30% weight)
  - % of daily targets completed this week (20% weight)
  - Average sessions per week over last 4 weeks (20% weight)
- Display the score prominently on the **Dashboard** as a large circular gauge (SVG arc) with:
  - Score number in the center
  - Color: Red (0–40), Amber (41–70), Green (71–100)
  - Label below: *"Consistency Score"*
  - Tooltip on hover: breakdown of each factor's contribution
- Also show score history as a small sparkline chart (last 30 days) below the gauge.
- Add score to `pst_consistency_history` array (one entry per day).
- Badge: **"Rock Solid"** 💎 — Maintain a score above 80 for 7 consecutive days.

### 15. ⚠️ Skill Decay Warning

- Track the **last practiced date** for each skill (already in skill object).
- If a skill has not been practiced in **5 or more days**:
  - The Skill Card visually **fades** (reduce opacity to 0.6, add a desaturated filter).
  - Show a pulsing amber warning badge on the card: *"⚠️ Getting Rusty!"*
  - If not practiced in **10+ days**, the badge turns red: *"🔴 Skill Decaying!"*
- On the dashboard, show a **"Skills Needing Attention"** section at the top (only visible when at least one skill is decaying) listing rusty skills with a "Practice Now" shortcut button.
- Send a browser notification (if enabled) when a skill hits 5 days without practice: *"⚠️ Your [Skill Name] skill is getting rusty! Log a session to keep it sharp."*
- In the **Progress page**, add a "Skill Health" section showing all skills color-coded by recency: Green (practiced within 2 days), Yellow (3–5 days), Red (5+ days).

- Add an optional **"Public Profile"** toggle in Settings.

- The public profile page (route: `/profile/:id`) shows a read-only view:
  - User name and avatar

  - Badge showcase (only unlocked badges)
  - Current streak and consistency score

### 18. 👥 Accountability Partner

- In Settings, add an **"Accountability Partner"** section.
- User can generate an **"Invite Link"** for their accountability partner.
- The invite link (`?partner=base64encodeddata`) encodes the user's:
  - Name and avatar
  - Current streak
  - Active skills list (names only)
  - Public profile ID
- When a partner opens the link:
  - They see a **"Connect as Accountability Partner"** page showing the inviter's stats.
  - They click **"Connect"** to save the partner's data locally in `pst_partner`.
- Once connected, the Dashboard shows an **"Accountability Partner"** widget card:
  - Partner's name and avatar
  - Their streak (last synced — shown as a snapshot from invite data, not live)
  - Last sync date
  - **"Send a Cheer 🎉"** button — copies a pre-written motivational message to clipboard: *"Hey! Just checking in — keep up your streak! 💪 Your accountability partner is rooting for you."*
  - **"Update Partner Data"** button — opens a text field to paste new partner data (partner re-shares their updated invite link).
- Note: This is intentionally offline/async — no real-time backend. Partners manually share updated links to sync. Make this clearly explained in the UI with a note: *"Sync manually by sharing your updated invite link."*
