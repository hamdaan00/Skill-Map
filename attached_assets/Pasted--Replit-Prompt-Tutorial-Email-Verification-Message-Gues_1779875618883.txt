## ✉️ FEATURE 2 — Clear Email Verification Message

When a user signs up with email + password, Supabase sends a verification email.
Right now users see nothing clear and get confused. Fix this with a beautiful,
informative confirmation screen.

### After Signup — Show a Dedicated Verification Screen

Instead of redirecting to dashboard or showing a vague message, replace the auth
form with a full **"Check Your Email"** screen:

**Design:**
- Large animated email icon: ✉️ (floating animation)
- Headline: **"Check your inbox! 📬"** (white, 28px, Sora bold)
- Subtext (clear and friendly):

  *"We've sent a verification link to:"*
  
  **[user@email.com]** (shown in a cyan pill/badge)
  
  *"Click the link in that email to activate your account and start tracking your skills."*

- A collapsible **"What if I don't see it?"** section:
  - *"Check your Spam or Junk folder"*
  - *"Make sure you typed your email correctly"*
  - *"The link expires in 24 hours"*

- **"Resend Email"** button:
  - Ghost style button
  - On click: calls `supabase.auth.resend({ type: 'signup', email })`
  - Shows a toast: *"✅ Verification email resent!"*
  - Disable button for 60 seconds after clicking to prevent spam
  - Show countdown: *"Resend again in 45s"*

- **"Wrong email? Go back"** link — returns to signup form

- A subtle animated background: slow moving dots or particles in dark navy

**Implementation:**
```javascript
// After supabase.auth.signUp() succeeds:
// Check if email confirmation is required
const { data, error } = await supabase.auth.signUp({ email, password });

if (data?.user?.identities?.length === 0) {
  // Email already registered
  setError("This email is already registered. Try logging in instead.");
} else if (data?.user && !data?.session) {
  // Email confirmation required — show verification screen
  setAuthStep("verify-email");
  setSubmittedEmail(email);
} else if (data?.session) {
  // Email confirmation disabled — go straight to app
  navigateToDashboard();
}
```

**After user clicks the verification link in their email:**
- Supabase redirects back to your app URL
- Detect the verified session:
```javascript
supabase.auth.onAuthStateChange((event, session) => {
  if (event === "SIGNED_IN" && session) {
    // User just verified — show welcome message
    setToast("🎉 Email verified! Welcome to SkillTracker!");
    navigateToDashboard();
  }
});
```

**In Supabase dashboard — update the redirect URL:**
1. Go to Supabase → Authentication → URL Configuration
2. Set **Site URL** to your Replit app URL (e.g. `https://skill-tracker.yourname.repl.co`)
3. Add to **Redirect URLs**: `https://skill-tracker.yourname.repl.co`

This ensures the verification link brings users back to YOUR app, not the Supabase default page.

## 👤 FEATURE 3 — "Continue as Guest" Button

Replace any "Skip for now" or "Skip" button on the auth/onboarding screen with a
clearly labeled **"Continue as Guest"** button.

### What Guest Mode Means
- User skips login entirely
- All data saves to localStorage only (no cloud sync)
- User can use the full app but data is local only
- A persistent subtle banner reminds them to create an account

### "Continue as Guest" Button Design
- Position: below the login/signup form, separated by a divider line with text **"or"**
- Style: ghost button, full width, grey text
- Label: **"Continue as Guest →"**
- Subtext below button (very small, muted): *"Your data will be saved on this device only"*

```jsx
<div style={{ textAlign: "center", margin: "16px 0" }}>
  <div style={{ display: "flex", alignItems: "center", gap: 12, marginBottom: 16 }}>
    <div style={{ flex: 1, height: 1, background: "rgba(255,255,255,0.08)" }} />
    <span style={{ fontSize: 12, color: "#475569" }}>or</span>
    <div style={{ flex: 1, height: 1, background: "rgba(255,255,255,0.08)" }} />
  </div>
  <button className="btn-ghost" style={{ width: "100%", padding: "12px" }}
    onClick={continueAsGuest}>
    Continue as Guest →
  </button>
  <p style={{ fontSize: 11, color: "#475569", marginTop: 6 }}>
    Your data will be saved on this device only
  </p>
</div>
```

### Guest Mode Banner
Once inside the app as a guest, show a **slim persistent banner** at the very top
of every page (below the navbar):

```
☁️  You're using SkillTracker as a guest. Create a free account to sync your data across devices.   [Create Account]
```

Banner design:
- Height: 36px, slim
- Background: `rgba(251, 191, 36, 0.08)` (subtle amber)
- Border-bottom: `1px solid rgba(251, 191, 36, 0.2)`
- Text: 12px, muted amber color `#d97706`
- **"Create Account"** link: cyan, underlined, opens auth modal
- A small **✕** to dismiss the banner for the current session
  (store `pst_banner_dismissed: true` in sessionStorage so it hides until next visit)

### Guest Mode Technical Implementation

```javascript
function continueAsGuest() {
  // Set guest flag
  localStorage.setItem("pst_is_guest", "true");
  // Remove any existing auth session
  // Navigate to onboarding (if first time) or dashboard
  const onboarded = localStorage.getItem("pst_onboarded");
  if (!onboarded) {
    navigateTo("onboarding");
  } else {
    navigateTo("dashboard");
  }
}

// Check if user is guest
function isGuest() {
  return localStorage.getItem("pst_is_guest") === "true" && !supabaseSession;
}
```

When a guest clicks **"Create Account"** from the banner:
- Open the auth modal/page
- After successful signup + verification, **migrate their localStorage data to Supabase**:

```javascript
async function migrateGuestDataToCloud(userId) {
  const guestSkills = loadData("pst_skills", []);
  const guestSessions = loadData("pst_sessions", []);

  if (guestSkills.length > 0) {
    // Upload skills to Supabase
    await supabase.from("skills").insert(
      guestSkills.map(skill => ({ ...skill, user_id: userId, id: undefined }))
    );
    // Upload sessions
    await supabase.from("sessions").insert(
      guestSessions.map(session => ({ ...session, user_id: userId, id: undefined }))
    );
    // Clear guest flag
    localStorage.removeItem("pst_is_guest");
    // Show migration success toast
    showToast("✅ Your guest data has been saved to your account!");
  }
}
```
